Microservices decomposition and identification

Microservice Backlog uses intelligent and genetic-programming techniques that calculates at design time each microservices' granularity. This model uses as inputs the user stories expressed in the product backlog, to decompose the functionalities or requirements of the application into microservices.

More information

User stories

Load the user stories of the project and model helps to define the size and number of microservices using artificial intelligence; it shows the microservices that are going to be part of an application detailing its dependencies and functionalities.

Metrics

The model calculates coupling, cohesion, and complexity metrics at design time. Therefore, we can observe and evaluate the microservices’ granularity and analyze how the application will be implemented and structured.

Intelligent algorithms grouping

The model uses intelligent algorithms that allows user stories or methods to be grouped into microservices. Such as a clustering algorithm (K-means, Scale Weighted K-means, bisecting k-means), genetic algorithm and a semantic similarity algorithm.

Microservices

From the model, the designer or developer can see the size of each microservice, as well as its complexity, dependencies, coupling, cohesion and development time.

Estimator of the use of computational resources

This defines the amount of hardware or software computational resources necessary to implement the candidates microservices depending on the architecture selected. When deployed in the cloud, this estimation can be translated into cost in money. Some of these resources may be: containers, CPU - Processing capacity, RAM, Use of hard disk, databases, etc
